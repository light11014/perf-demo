# 프론트엔드 성능 병목 데모 프로젝트

이 프로젝트는 일반적인 프론트엔드 성능 문제(Anti-Patterns)를 직접 체험하고 최적화 방법을 학습할 수 있도록 제작된 Next.js 기반의 실습용 애플리케이션입니다.

## 프로젝트 실행 방법

```bash
# 1. 의존성 설치
npm install

# 2. 개발 서버 실행
npm run dev
```
브라우저에서 `http://localhost:3000` 주소로 접속하여 각 데모 페이지를 확인할 수 있습니다.

## 데모 페이지 상세 설명

각 페이지는 특정한 성능 문제를 의도적으로 가지고 있습니다. Chrome 개발자 도구(F12)의 Network, Performance 패널 등을 함께 활용하면 문제점을 더 명확하게 분석할 수 있습니다.

| 카테고리 | 페이지 경로 | 문제점 설명 | 주요 관련 지표 | 분석 상태 |
| :--- | :--- | :--- | :--- | :--- |
| **번들** | `/bundle/heavy-import` | `lodash` 라이브러리 전체를 가져와 불필요하게 자바스크립트 번들 크기를 증가시킵니다. | `FCP`, `LCP` | **정상 작동** |
| **번들** | `/bundle/no-split` | 탭 UI에 필요한 모든 컴포넌트를 코드 분할 없이 한 번에 로드하여 초기 로딩을 저해합니다. | `FCP`, `LCP` | **정상 작동** |
| **인터랙션**| `/interaction/long-task` | 입력 시마다 무거운 동기적 계산을 실행하여 메인 스레드를 차단하고 UI 반응성(INP)을 저해합니다. | `INP` | **정상 작동** |
| **인터랙션**| `/interaction/rerender` | 검색창 입력에 따른 상태 변경 로직이 없어, 의도된 '불필요한 리렌더링' 문제가 발생하지 않습니다. | `INP` | **수정 필요** |
| **네트워크**| `/network/no-cache` | API 응답에 캐시 정책이 없어, 페이지 이동 시마다 동일한 데이터를 반복적으로 요청합니다. | - | **정상 작동** |
| **네트워크**| `/network/waterfall` | 여러 API 요청을 병렬이 아닌 순차적으로 처리하여 전체 데이터 로딩 시간을 지연시킵니다. | `LCP` | **정상 작동** |
| **렌더링** | `/rendering/blocking-css` | 여러 폰트 CSS를 `<head>`에서 동기적으로 로드하여 첫 콘텐츠 렌더링(FCP)을 차단합니다. | `FCP`, `LCP` | **정상 작동 (수정됨)** |
| **렌더링** | `/rendering/cls` | 이미지 공간을 미리 확보하지 않고 뒤늦게 로드하여, 레이아웃을 밀어내는 CLS를 유발합니다. | `CLS` | **정상 작동** |
| **렌더링** | `/rendering/font-loading` | `font-display: block` 옵션으로 인해 폰트 로딩 중 텍스트가 보이지 않는 현상(FOIT)을 발생시킵니다. | `FCP`, `LCP` | **정상 작동** |
| **렌더링** | `/rendering/multiple-images` | 다수의 이미지를 비효율적인 `background-image` 방식으로 로드하여 네트워크 병목을 유발합니다. | `LCP` | **정상 작동** |
| **렌더링** | `/rendering/slow-lcp` | 페이지의 핵심 요소인 거대한 이미지를 최적화나 우선순위 지정 없이 로드하여 LCP를 지연시킵니다. | `LCP` | **정상 작동** |
| **렌더링** | `/rendering/unoptimized-landing` | 느린 LCP, 최적화되지 않은 여러 이미지 등 다양한 문제를 종합적으로 보여주는 랜딩 페이지입니다. | `LCP` | **정상 작동** |
